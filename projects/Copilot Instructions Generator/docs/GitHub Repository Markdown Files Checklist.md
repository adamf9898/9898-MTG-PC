# GitHub Repository Markdown Files Checklist

---


Based on standard GitHub repository practices, you should have the following Markdown files (with recommended filenames):

---


- [ ] README.md
- [ ] CONTRIBUTING.md
- [ ] CODE_OF_CONDUCT.md
- [ ] LICENSE or LICENSE.md
- [ ] SECURITY.md
- [ ] PULL_REQUEST_TEMPLATE.md
- [ ] ISSUE_TEMPLATE.md

---

- [ ] CHANGELOG.md

## Markdown Files Table Checklist

| Checklist | File | Description |
|-----------|------|-------------|
| [ ] | [README.md](../README.md) | Project overview, features, setup, and usage instructions. |
| [ ] | [CONTRIBUTING.md](../CONTRIBUTING.md) | Guidelines for contributing, coding standards, and pull request process. |
| [ ] | [CODE_OF_CONDUCT.md](../CODE_OF_CONDUCT.md) | Community standards and expected behavior. |
| [ ] | [LICENSE](../LICENSE) or [LICENSE.md](../LICENSE.md) | Project license (MIT, Apache, etc.). |
| [ ] | [SECURITY.md](../SECURITY.md) | Security policy and vulnerability reporting instructions. |
| [ ] | [PULL_REQUEST_TEMPLATE.md](../PULL_REQUEST_TEMPLATE.md) | Pull request template for contributors. |
| [ ] | [ISSUE_TEMPLATE.md](../ISSUE_TEMPLATE.md) or .github/ISSUE_TEMPLATE/ | Templates for bug reports and feature requests. |
| [ ] | [Discussions.md](../Discussions.md) | (Optional) For community discussion topics if not using GitHub Discussions. |
| [ ] | [CHANGELOG.md](../CHANGELOG.md) | (Optional) Record of changes and version history. |
| [ ] | [ROADMAP.md](../ROADMAP.md) | (Optional) Future plans and milestones for the project. |
| [ ] | [FAQ.md](../FAQ.md) | (Optional) Frequently asked questions about the project. |
| [ ] | [SUPPORT.md](../SUPPORT.md) | (Optional) Information on how to get support for the project. |
| [ ] | [ARCHITECTURE.md](../ARCHITECTURE.md) | (Optional) Overview of the project's architecture and design decisions. |
| [ ] | [TESTING.md](../TESTING.md) | (Optional) Instructions for running tests and contributing tests. |
| [ ] | [DEPLOYMENT.md](../DEPLOYMENT.md) | (Optional) Deployment instructions and environment setup. |
| [ ] | [GOVERNANCE.md](../GOVERNANCE.md) | (Optional) Project governance model and decision-making process. |
| [ ] | [MAINTAINERS.md](../MAINTAINERS.md) | (Optional) List of project maintainers and their roles. |
| [ ] | [SUPPLEMENTARY.md](../SUPPLEMENTARY.md) | (Optional) Any additional documentation relevant to the project. |

---


Use this checklist to ensure your repository is well-documented and follows best practices for open-source projects on GitHub.
